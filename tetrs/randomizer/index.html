Module tetrs::randomizer [−] [src]

Implements a randomizer.

Randomizers currently only are required to implement the Iterator trait. Preview pieces are managed by the caller, although this could be something added as a trait.

It is arguable whether Randomizer should imply Iterator. A Randomizer is often only processed a single element at a time, and cannot be used in an iterator fashion due to borrow rules.

Also, all Randomizer's should return infinite sequences so we can remove the required unwrap on manual calls to next.
